# Декодирование Unicode-hex токенов в именах звуков (только отображение)

**Дата:** 2026-07-20
**Статус:** утверждён к реализации

## Проблема

В именах файлов встречаются декоративные символы, закодированные как текст:
`#$2731 bass fuckemup.wav`. Пользователь видит «сырой» набор `#$2731` вместо
символа. `$2731` — это hex-код-поинт Unicode: `0x2731` → `✱` (heavy asterisk).
Такие символы (✱ ✦ ❤ ★ ♪ и эмодзи) продюсеры кладут в названия паков; где-то
по пути они превратились в escaped-текст.

Нужно: при **отображении** в интерфейсе показывать настоящий символ. Файлы на
диске и данные (`sample.name`) не трогаем — переименования нет.

## Область

- **Только отображение** в UI. Никаких изменений на диске, в backend, в данных.
- Поверхности: список звуков (`SoundTable`) и плеер (`PlayerPage`).

## Решение

### Чистый хелпер: `frontend/src/shared/lib/decodeSampleName.ts`

```ts
export function decodeSampleName(name: string): string
```

- **Паттерн:** `#?\$([0-9A-Fa-f]{4,6})` — необязательная `#`, обязательный `$`,
  затем 4–6 hex-цифр. Флаг `g` — несколько кодов в одном имени
  (`#$2731#$2764` → `✱❤`).
- **Замена:** каждый токен → `String.fromCodePoint(parseInt(hex, 16))`.
- **Почему `$` + минимум 4 hex:** отсекает ложные срабатывания. Цена вроде
  `$50 loop.wav` (2 цифры) не трогается; декоративные символы живут в
  `0x2600–0x2BFF`, эмодзи в `0x1F300+` — всегда 4–5 hex-цифр.
- **Безопасность:** `String.fromCodePoint` в `try/catch`; при невалидном
  код-поинте (> 0x10FFFF и т.п.) оставляем исходный токен без изменений.
- Идемпотентность не требуется, но функция чистая и без побочных эффектов.

### Тест: `frontend/src/shared/lib/decodeSampleName.test.ts`

По образцу `ytKeywords.test.ts`. Кейсы:
- `#$2731 bass fuckemup.wav` → `✱ bass fuckemup.wav`
- `$2731 loop.wav` → `✱ loop.wav` (без `#`)
- `#$2731#$2764 x` → `✱❤ x`
- `$50 loop.wav` → без изменений (2 цифры)
- имя без токенов → без изменений
- невалидный код (`$FFFFFF`... > 0x10FFFF) → токен оставлен как есть

### Проводка (только видимые имена)

- `SoundTable.tsx:270` — `{decodeSampleName(sample.name)}`.
- `PlayerPage.tsx` — видимые заголовки треков: `entry.name` (строки 574, 663) и
  `e.name` (строка 915) оборачиваем в `decodeSampleName(...)`. Атрибуты `title`
  можно оставить как есть (там подсказки, не имя).

## Вне области (YAGNI)

- Переименование файлов на диске.
- Декодирование в backend / при экспорте / в YouTube-описаниях.
- Другие форматы escaping (`U+`, `\u`, `&#x…;`) — не встречались в примерах.

## Критерии готовности

- Юнит-тесты проходят.
- В таблице и плеере `#$2731 …` отображается как `✱ …`.
- Обычные имена без токенов не меняются; цены `$50` не ломаются.
