# Перекройка диалога публикации на YouTube

**Дата:** 2026-07-17
**Статус:** согласовано

## Проблема

Диалог публикации вырос органически: одна колонка шириной 580px и высотой под 2000px, где подряд свалены обложка, превью, шрифт, ник, авторы, шаблоны, теги, приватность и список видео. Компонент `YtUploadDialog` — 1060 строк (`PlayerPage.tsx:1333-2392`).

Жалобы пользователя, дословно: «очень неудобно и непонятно, особенно когда постишь сразу много», «очень громоздко, всё подряд везде разбросано, глаза бегают туда-сюда».

Разобранные по частям:

1. **Нет обзора.** Когда постишь пачку, не видно разом: какие обложки, какие названия, где что поправлено, что уже загрузилось.
2. **Непонятна область действия.** Общее для пачки и своё для видео перемешано и разнесено по разным концам скролла.
3. **Узкая колонка.** 580px и бесконечная прокрутка.
4. **Беготня глаз.** Превью вверху, список видео внизу — а они про одно и то же.

Шаблоны `{name}` / `{bpm}` пользователя устраивают и **не трогаются**.

### Отдельно: авторы

Авторы распознаются из имени каждого файла, то есть это свойство видео. Текущее состояние:

- при загрузке авторы **уже** берутся по-битово (`beatAuthors(b, ...)` внутри `start()`), так что опубликованное не врёт;
- но чипсы авторов в UI намертво прибиты к первому биту — `beats[0]` в коде (`PlayerPage.tsx:1542-1543`), с комментарием «Чипсы авторов по-прежнему берут первый бит». Увидеть и поправить авторов остальных видео нельзя;
- **баг:** ручные добавления авторов (`extras: string[]`) общие на пачку и приклеиваются ко всем битам сразу (`beatAuthors` дописывает `extras` каждому). Добавил соавтора одному — получили все.

## Решение

### Раскладка: две панели

Окно ~1000px (`maxWidth: 94vw`) вместо 580px.

```
┌─────────────────────────────────────────────────┐
│ Публикация на YouTube          6 видео · 1 правлено │
├───────────────┬─────────────────────────────────┤
│ ВСЕ ВИДЕО  🎲 │ [Это видео] [Общее для всех]    │
│ ▸ обложка     │ ┌──────────┬──────────────────┐ │
│   название    │ │  кадр    │ авторы этого бита│ │
│   авторы   ●  │ │ превью   │ название         │ │
│ ─────────────  │ │ обложка… │ описание         │ │
│   …ещё 5      │ │  🎲  ▶   │ теги / доступ    │ │
│               │ └──────────┴──────────────────┘ │
├───────────────┴─────────────────────────────────┤
│ ● Канал подключён · ffmpeg найден  [Закрыть][↑6]│
└─────────────────────────────────────────────────┘
```

**Левая панель** — все видео сразу: обложка, название (итоговое, из резолвера), авторы этого бита, точка «правлено руками», процент загрузки. Над списком — «🎲 Перетасовать» (действует на список, поэтому живёт над ним).

**Правая панель** — вкладки, называющие область действия вслух:

- **«Это видео»** — кадр превью, выбор обложки, 🎲, «Превью видео», авторы бита, название, описание, теги, доступ.
- **«Общее для всех»** — шаблон названия, шаблон описания, ник продюсера, теги по умолчанию, доступ по умолчанию, текст на кадре + шрифт.

**Подвал** — статус канала и ffmpeg (сейчас плашка посреди формы) + «Закрыть» / «Загрузить N видео».

**Одиночное видео** (не пачка): левый список и вкладки прячутся, остаётся одна колонка — «общее» и «это видео» там совпадают, и различать нечего.

### Авторы становятся свойством видео

- `extras: string[]` → `extras: Record<string, string[]>` по пути бита.
- `beatAuthors(b, nick, aliases, extras[b.path] ?? [])` — добавления больше не текут на всю пачку.
- `chips` считаются от `focusBeat`, а не от `beats[0]`.
- `aliases` (переименования) **остаются общими**: это словарь опечаток, «hoodrunah → hoodrunnah» правильно чинить один раз и везде.
- Ник продюсера остаётся общим.

### Разделение компонента

`YtUploadDialog` (1060 строк) делится по границам панелей — иначе править раскладку в монолите слишком рискованно:

- `YtUploadDialog` — состояние, резолвер, загрузка, каркас;
- `VideoList` — левая панель;
- `ThisVideoPane` — вкладка «Это видео» (включает уже написанные поля из `BeatEditor`, который в неё вливается);
- `SharedPane` — вкладка «Общее для всех»;
- `CoverPicker` — панель поиска обложек (сейчас вложена в форму).

Состояние остаётся в `YtUploadDialog` и передаётся пропсами: панели остаются глупыми и тестируемыми глазами по отдельности.

## Не входит

- Шаблоны и их синтаксис — не трогаем.
- Логика рендера кадра и загрузки — не трогаем, только переезжает вызов.
- Персистентность правок между открытиями диалога — как и раньше, только на сессию.

## Критерии готовности

1. Пачка: все видео видны слева сразу, с обложками, названиями и авторами каждого.
2. У разных битов в списке показаны разные авторы, соответствующие их именам файлов.
3. Добавление автора одному видео не появляется у остальных.
4. Переименование автора применяется ко всем битам, где встречается тот же токен.
5. Вкладка «Общее для всех» меняет все видео, кроме полей, правленых руками.
6. Одиночное видео: ни списка, ни вкладок — одна колонка.
7. Загрузка уезжает с теми значениями, что показаны в диалоге.
