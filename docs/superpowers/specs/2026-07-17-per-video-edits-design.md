# Правка отдельных видео в пачке

**Дата:** 2026-07-17
**Статус:** согласовано

## Проблема

При загрузке пачки через кнопку YouTube поля «Название», «Описание», «Теги» и «Приватность» — общие шаблоны на всю пачку. Название каждого видео генерируется из шаблона (`renderYtTemplate(tpl, b, nick, authors)`, `PlayerPage.tsx:2221`), поэтому поправить текст одного конкретного видео нельзя: правка шаблона немедленно переписывает все.

Переключение между видео уже работает — клик по строке списка ставит `focus` (`PlayerPage.tsx:2188`), — но меняет только кадр превью и адресата выбора обложки, а не редактируемые поля.

## Решение

### Модель

Карта переопределений, живущая в состоянии диалога (как `extras`: она относится к конкретной пачке, в настройках ей не место):

```ts
type BeatEdit = { title?: string; desc?: string; tags?: string; privacy?: string };
const [edits, setEdits] = React.useState<Record<string, BeatEdit>>({});
```

`undefined` у поля = наследуется из шаблона. Заданное значение = своё. Благодаря этому последующая правка шаблона переписывает только ненаследованные поля — в этом весь смысл переопределения.

### Резолвер

Единственный источник значений видео:

```ts
const resolveBeat = (b: YtBeat) => {
  const e = edits[b.path] ?? {};
  const authors = beatAuthors(b, nick, aliases, extras);
  return {
    title: e.title ?? renderYtTemplate(tpl, b, nick, authors),
    description: e.desc ?? renderYtVars(desc, b, nick, authors),
    tags: e.tags ?? tags,
    privacy: e.privacy ?? privacy,
  };
};
```

Потребители:

- строка списка — показывает `title` (сейчас зовёт `renderYtTemplate` напрямую);
- `start()` при загрузке (`PlayerPage.tsx:1704-1707`) — сейчас прогоняет шаблон вслепую, из-за чего переопределения не доехали бы до YouTube.

### Интерфейс

Выбранная строка списка раскрывается в форму: «Название» (input), «Описание» (textarea), «Теги» (input), «Приватность» (select). У каждого поля кнопка ↺ — активна только когда поле переопределено, возвращает его к шаблону.

На строке — точка-маркер, если у видео есть свои правки: видно, не раскрывая.

Список подрастает со 180px до 320px, иначе раскрытая форма не помещается.

Клики по обложке и 🎲 раскрытие не триггерят — там уже стоит `stopPropagation`.

## Границы

Инлайн-редактор только для пачки (`isBatch`). Для одиночного видео поля наверху и так правят именно его; второй способ был бы путаницей.

Переопределения не персистятся между открытиями диалога — они про конкретную пачку.

## Критерии готовности

1. Клик по любой строке раскрывает форму именно этого видео.
2. Правка названия одного видео не трогает остальные.
3. Правка шаблона после этого переписывает только те видео, где поле не трогали.
4. ↺ возвращает поле к шаблону и снова подчиняет его шаблону.
5. Загруженное на YouTube видео получает переопределённые значения, а не шаблонные.
