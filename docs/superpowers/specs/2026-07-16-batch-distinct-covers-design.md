# Разные обложки на пачку (по тайп-артисту)

**Дата:** 2026-07-16 · **Статус:** approved (Вариант 1) · **Область:** Player → диалог загрузки на YouTube

## Задача
При загрузке пачки видео каждому биту нужна **своя** обложка, но все — по **тайп-артисту** бита. Сейчас на всю пачку одна картинка (`image`).

## Решения (брейншторм)
- **Авто-подбор + обзор с заменой** (не полностью авто, не вручную каждую).
- Замена per-beat: **и 🎲** (следующая неиспользованная из пула), **и клик по обложке → Pinterest-пикер** для этого бита. Плюс «перетасовать все».
- **Вариант 1:** пул по тайпу + ленивое скачивание.

## Модель
- `pools: Record<typeName, CoverImage[]>` — перемешанный пул обложек на каждый уникальный тайп пачки (поиск `«{тайп} aesthetic»`, ~40).
- `assign: Record<beatPath, CoverImage>` — назначенная (пока НЕ скачанная) обложка бита.
- `focus: beatPath` — какой бит показан в большом 16:9-превью с текстом (по умолчанию первый).
- `pin.target?: beatPath | null` — режим пикера: назначить конкретному биту (иначе — старое поведение единой `image`).
- Ничего не персистится (сессия диалога).

## Поток
1. Открыли пачку (>1 бита): `autoAssign()` — для каждого уникального `typeName` один раз тянем пул (`/api/covers/search?q={тайп} aesthetic&limit=40`), перемешиваем, раздаём **разные** битам этого тайпа по кругу. Тайпы разные → каждый бит матчит свой.
2. **Превью — по удалённому `thumb`** (webview и так грузит их в Pinterest-гриде); ничего не качаем.
3. Обзор: в списке битов у каждого — миниатюра (`assign[path].thumb`) + 🎲 + клик по ней → пикер для этого бита; сверху кнопка «перетасовать все».
4. **Скачиваем полную картинку только на загрузке** (`/api/covers/download` для того бита, что уходит) → `imagePath`.

## Алгоритмы
- `autoAssign()` / `reshuffleAll()`: `ensurePools()` → для каждого тайпа `shuffle(pool)`, битам этого тайпа `pool[i % pool.length]` (i — индекс среди битов тайпа). Повтор только если битов больше, чем картинок.
- `cycleBeat(path)`: следующая картинка пула тайпа (курсор + 1), не равная текущей.
- `pickForBeat(path)`: открыть `pin` с `target=path`, `q={тайп} aesthetic`.
- `pinPick(img)`: если `pin.target` → `assign[target]=img`, закрыть; иначе старое (скачать → `image`).

## Загрузка (`start`)
Для каждого бита: если есть `assign[path]` → `imagePath = (await coversDownload(assign[path].full)).path`; иначе `image` (одиночный/фолбэк). Дальше как сейчас (рендер+аплоад с наложением/авторами).

## UI
- **Пачка (>1):** прячем одиночные контролы обложки; показываем блок «Обложки (авто, разные)» + кнопка «перетасовать все». Большое 16:9-превью с текстом — для `focus`-бита (клик по строке бита фокусит). В списке битов: [миниатюра+🎲] [название] [статус].
- **Один бит:** без изменений (текущий `image`/пикер/превью).

## Крайние случаи
- Нет `typeName` у бита → авто-обложку не назначаем (останется одиночная `image`/ручной выбор).
- Пул пустой (поиск ничего не дал) → бит без обложки, грузить нельзя, подсказка.
- Пул меньше числа битов → повтор по кругу (иначе никак).
- Один бит в пачке → старый флоу.

## Интеграционные точки
- `frontend/src/pages/player/PlayerPage.tsx` — `YtUploadDialog`: состояние `pools/assign/focus`, `autoAssign/reshuffleAll/cycleBeat/pickForBeat`, `pin.target`, per-beat миниатюры в списке, ветвление `start()` по обложке, превью по `focus`.
- API не меняется (используем существующие `/api/covers/search` и `/api/covers/download`).
- i18n `ru/en` — метки (перетасовать, «другая», обложки авто).

## Вне области
Кэш/персист пулов; ручная правка запроса пула per-type (берём `«{тайп} aesthetic»`); дедуп одинаковых картинок между разными тайпами.
