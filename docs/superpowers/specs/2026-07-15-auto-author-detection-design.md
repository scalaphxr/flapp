# Авто-определение авторов бита при выкладке на YouTube

**Дата:** 2026-07-15 · **Статус:** approved (Вариант 1) · **Область:** вкладка Player → диалог загрузки на YouTube

## Задача
Многие биты сделаны 2+ продюсерами; авторы уже есть в имени файла, но у всех **разные форматы** имён. Нужно автоматически распознавать **всех** авторов и подставлять их при выкладке.

## Решения (из брейншторма)
- **Куда:** переменная `{authors}` (для шаблонов названия/описания) **+** строка «prod.» на видео-надписи. (Теги — нет.)
- **Подача:** редактируемые **чипсы** в диалоге + **память псевдонимов** (правки запоминаются и применяются в следующий раз).
- **Свой ник** (`ytNickname` из настроек): **всегда в списке и первым** (добавляется, если в имени его нет).
- **Мультипользовательность:** никакого хардкода конкретного ника; свой ник берётся из настроек.
- **Подход:** эвристический парсер по правилам (офлайн, детерминированный). LLM и теги — вне области v1.

## Сигналы парсинга (подтверждены на 8 реальных примерах)
- `@handle` — **самый надёжный** маркер автора (соцсети), где угодно в имени.
- Разделители соавторов: `@`, `+`, `&`, `x`, `–`/`-`, `,`, пробел, а также **слитно** (`success135nsm`).
- BPM в видах: `144`, `142bpm`, `113 BPM`, `(167)`, прилипший `136@mdm`.
- Ведущее **4-значное** число (`0019`, `1161`) = индекс/каталог → отбросить.
- **Имя бита** (`holyshi`, `GLAMOUR`, `Space Cadet`) идёт ДО bpm → в авторы НЕ идёт.
- Шум-слова: `bpm`, `prod`, `p`, `type`, `beat`, `free`, `exclusive`, `feat/ft`, тональности (`bmin`, `f min`, `amin`), числа.
- Авторы обычно **после** bpm, но иногда **до** имени (кейс `@a + @b – name – bpm`) — спасают `@`-хендлы.

## Конвейер `parseAuthors(stem, { nick, aliases, bpm }) → string[]`
1. Вытащить все `@([a-z0-9_]+)` как гарантированных авторов (запомнить порядок).
2. Расклеить границы цифра↔буква (`success135nsm` → `success 135 nsm`, `142bpm` → `142 bpm`).
3. Токенизировать по пробелам и `+ & , / – — -`; `x` — разделитель только как отдельный токен.
4. Найти **BPM-якорь**: токен == round(analyzedBpm), иначе первое `^\d{2,3}$` в 40–260 (не ведущий 4-значный индекс).
5. Отбросить ведущий 4-значный индекс.
6. **Регион авторов** = (все `@`-хендлы) ∪ (токены-хвост ПОСЛЕ bpm). Токены ДО bpm — это имя, отбрасываются.
7. Отфильтровать шум: набор шум-слов, тональности (`^[a-g](#|b)?(m|min|maj|minor|major)?$` и одиночные `min/maj/…`), чистые числа, `p`/`prod`.
8. Нормализовать: снять `@`, привести к нижнему регистру (конвенция type-beat).
9. Применить **карту псевдонимов** `aliases[token] → canonical` (пустая строка = «не автор», выкинуть).
10. Дедуп (без регистра, с сохранением порядка). Prepend `nick`, если его нет → всегда первый.

## Формат вывода
- `{authors}` = `authors.join(" x ")` → напр. `flipgod x drowninstatic x nsm`.
- Строка «prod.» на видео = `prod. ` + тот же join (заменяет прежнее `prod. {nick}`).
- `{nick}` остаётся отдельно (только свой ник).

## Данные
- **Backend (persist-only):** `Settings.YtAuthorAliases map[string]string` (`json:"ytAuthorAliases"`). Парсинг — на фронте; бэкенд только хранит/отдаёт карту. `mergeDefaults`: nil → пустая карта.
- Свой ник — существующее `YtNickname`.

## UI (диалог загрузки)
- Блок **«Авторы (авто)»** над/рядом с ником: чипсы = распознанные авторы **репрезентативного** (первого) бита.
- Каждый чип: инлайн-input (переименование) + «×» (удалить). Внизу — input «+ добавить».
- Захват правок:
  - переименовал чип → `aliases[originalToken] = newValue` (глобально, persist);
  - удалил распознанный чип → `aliases[originalToken] = ""` (drop, persist);
  - добавил вручную → session-список `extraAuthors` (применяется ко всем битам пачки, не persist);
  - удалил добавленный → убрать из `extraAuthors`.
- Живой предпросмотр (обложка/видео, `{authors}`) реагирует на изменения.
- Кнопка «сброс» — перечитать из имени (сбросить extraAuthors; алиасы остаются).

## Пачки
Каждый бит парсится независимо (аліасы применяются ко всем). Чипсы редактируют первый бит; rename/drop → глобальные алиасы (влияют на всю пачку), ручные добавления `extraAuthors` — на всю пачку (сессия). Ограничение задокументировано.

## Интеграционные точки

- `frontend/src/pages/player/PlayerPage.tsx` — состояние алиасов/extra/чипсов; `overlayLinesFor` и `renderYtVars` получают авторов; `{authors}`; чипсы-UI; проброс в upload/preview; persist алиасов.
- `frontend/src/shared/api/types.ts` + `shared/model/settings.ts` — `ytAuthorAliases`.
- `backend/internal/infrastructure/settings/settings.go` — поле `YtAuthorAliases`.
- i18n `ru/en` — метки.

## Тестирование
Чистый `parseAuthors` покрыть Node-проверкой на 8 подтверждённых примерах как ground truth:
```
0019 holyshi 144 wwra                         → [nick?, wwra]  (0019 индекс, holyshi имя)
success135northsidemiracle                    → [northsidemiracle]
SUCCESS 113 BPM DROWNINSTATIC NSM             → [drowninstatic, nsm]
1161 GLAMOUR P KOLALOI+WSB+TAINIY 180 BMIN    → [kolaloi, wsb, tainiy]   (P=prod)
space 6 136@mdm @wh1sper                       → [mdm, wh1sper]
@tailsxm + @onest2k – aurora - 137 bpm f min   → [tailsxm, onest2k]
GasStation 142bpm @damnflaryxx @djfrogprod @may→ [damnflaryxx, djfrogprod, may]
(167) Space Cadet @sxldner                     → [sxldner]
```
(при пустом `nick` — без своего ника; иначе `nick` первым.)
